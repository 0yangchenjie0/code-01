1.注释
  注释是 程序员 在 源代码中 添加的对代码功能解释说明的文字，是用来帮助程序员理解程序的（给人看的）。
  三种类型的注释：
	  单行注释  
		// 注释的内容
	  多行注释
		/*
		  注释的内容
		  注释的内容 可以有多行
		  
		  /*
			错误：多行注释不能嵌套
		  */
		*/
		多行注释不能嵌套
	  文档注释  用来生成帮助文档
		/**
		  内容可以是一行到多行
		*/
  注释的特征：注释是在源代码中存在，给程序理解程序用，注释不会编译到字节码文件中。
2.变量
    数据类型 变量名称 = 变量内容;
    int age = 18; // 声明变量并赋值
    age = 20; // 改变变量的值
    注意事项：
    变量的特点：在于“变”字，声明变量所开辟的内存空间可以多次赋值。
	1.在方法中声明的变量，必须赋值之后才能使用。
	2.先声明后赋值，没用声明的变量名什么都不是。
	3.赋值的数据类型必须和声明变量的数据类型一致。
	4.在同一个方法中，不能声明多个同名的变量。
3.常量（constant）
    常量的内容不可以更改
    在java语言中，使用 final 关键字声明常量
    final 数据类型 常量名称 = 常量的值;
    // 声明常量并赋值
    final double PI = 3.1415926;
    【注意事项】
	1.常量只能被赋值一次，赋值完毕后就不能再更改值。
	2.常量名一般由大写字母组成，多个单词可以用下划线分割，例如：final int MAX_VALUE = 100;
4.关键字
   public class HelloWorld {
		public static void main(String[] args) {
		}
   }
   关键字：public、class、static、final、void、int、double、char
   什么是关键字：关键字是被 Java 语言赋予特殊含义，具有专门用途的单词，不能再做其它使用。
   特点：
	 每个关键字有专门用途（不能再用于其它用途）
	 关键字全部是小写
	 变量名称、常量名称、方法名称、类名不能使用关键字
5.标识符
  标识符是程序员在编写代码的时候，给 变量、常量、类、方法.... 起的名称。
  1.标识符的命名规则
    a.标识符有 字母（a-zA-z）、数字（0-9）、下划线(_)、美元符号($) 组成
    b.标识符不能以数字开头
    c.标识符不能使用关键字
    int $a = 100; // 正确
    int 100Hello = 200; // 错误不能以数字开头
    int student_age = 300;// 正确
    int num = 200; // 正确
    int @a = 300; // 错误 @不能作为标识符的字符
    int AGE = 200; // 正确
  2.标识符的命名规范
    养成良好的编码习惯
    标识符要见名知意，简洁
    1.变量、方法名的命名规范 小驼峰命名
       首字符小写，多个单词，后面单词首字母大写
       int orderNumber;
       String username;
       public void hello() {
       }
       public void sayHello() {
       }
    2.类的命名规范，大驼峰命名
		public class HelloWorld {
		}
		public class Hello {
		}
   注意：
     Java不采用通常语言使用的ASCII字符集，而是采用 unicode 这样的标准的国际字符集。
     因此，这里的字母的含义不仅仅是英文，还包括汉字等等。但是不建议大家使用汉字来定义标识符！
     标识符可以使用汉字，但是不建议这么做
